# Story 2.2 — Extração via Gemini Vision

## Status: Done

**Linear:** BLA-42 | **Completed:** 2026-03-25

## Story

**Como** sistema,
**Quero** extrair os dados de um documento médico via Gemini 2.5 Flash,
**Para** que as informações fiquem estruturadas e prontas para análise.

---

## Acceptance Criteria

### AC1 — Extração de documentos
- PDFs: texto extraído via `pdf-parse`, enviado como texto ao Gemini
- Imagens (JPG/PNG): enviadas diretamente ao Gemini como conteúdo multimodal
- Documentos >200k chars truncados com aviso incluído no output
- `temperature: 0.1` para máxima precisão numérica

### AC2 — Output estruturado via AI SDK v6
- Output via `generateText` + `Output.object()` com schema Zod
- JSON válido garantido pelo schema Zod — sem parsing manual
- Model: `google/gemini-2.5-flash` via `@ai-sdk/google`
- `GOOGLE_GENERATIVE_AI_API_KEY` no `.env`

### AC3 — Schema Zod de output
- `MedicalDocumentSchema` com campos: documentType, examDate, overallSummary, patientInfo (sem PII), providerInfo, modules[]

### AC4 — Sanitização de PII (LGPD)
- Após extração: campos PII removidos antes de qualquer retorno
- Schema Zod não inclui campos PII — prevenção em dupla camada

### AC5 — Fallback estruturado
- Em caso de erro: retornar objeto com `documentType: "UNKNOWN"`, `overallSummary: "Não foi possível extrair os dados"`, `modules: []`

### AC6 — Função exportada
- `lib/documents/extractor.ts` exporta `extractMedicalDocument(content, fileName, mimeType)`
