## Story

**Como** desenvolvedor,
**Quero** ter o projeto Next.js 16 configurado com Drizzle ORM, Neon Postgres e pgvector,
**Para** que todos os épicos possam ser desenvolvidos sobre uma base sólida.

---

## Contexto

Setup fundacional do SAMI. O banco Neon Postgres já está provisionado (`DATABASE_URL` no `.env`). Mobile-first PWA com dark mode padrão.

---

## Acceptance Criteria

### AC1 — Inicialização do projeto
* Next.js 16, TypeScript strict, App Router, `src/` directory
* `pnpm` como package manager
* Shadcn UI com tema `zinc`, dark mode padrão (classe `dark` no `<html>`)
* Tailwind CSS v4 configurado
* Geist Sans + Geist Mono via `next/font`
* `pnpm dev` roda sem erros em `localhost:3000`

### AC2 — Drizzle ORM + Neon Postgres
* `drizzle-orm`, `@neondatabase/serverless`, `drizzle-kit` instalados
* `lib/db/client.ts` exporta `db` singleton usando `neon()` driver
* `lib/db/schema/index.ts` exporta todos os schemas
* Scripts: `db:generate`, `db:migrate`, `db:push`, `db:studio`, `db:seed`, `db:setup`

### AC3 — pgvector habilitado
* Migration: `CREATE EXTENSION IF NOT EXISTS vector`
* Schema usa tipo `vector(768)` para colunas de embeddings

### AC4 — Schema base (tabelas)
* `users`: id (UUID), email (unique), passwordHash, role (enum: patient/admin), onboardingCompleted (bool), isActive (bool), createdAt, updatedAt
* `medical_profiles`: id, user_id (FK unique), campos básicos, createdAt, updatedAt
* `password_reset_tokens`: id, userId (FK), token (unique), expiresAt, usedAt, createdAt
* Migration aplicada sem erros

### AC5 — PWA manifest
* `public/manifest.json` com name, short_name, display: "standalone", theme_color, icons
* Meta viewport mobile em `app/layout.tsx`

### AC6 — Estrutura de pastas
```
app/(auth)/   # login, registro, reset
app/(app)/    # rotas protegidas
app/api/
lib/db/client.ts
lib/db/schema/index.ts
lib/auth/config.ts
components/ui/
tests/unit/
tests/integration/
tests/e2e/
```

### AC7 — Vitest smoke test
* `tests/unit/smoke.test.ts` com 1 teste passing
* Se Story 0.4 já configurou Vitest, apenas adicionar o teste

- Next.js 16 não tem `next lint` — substituído por `eslint src` com flat config nativo ESLint 9

- pgvector adicionado manualmente ao início da migration gerada pelo Drizzle
- Icons PWA (icon-192.png, icon-512.png) requerem assets de design — placeholders nos paths
