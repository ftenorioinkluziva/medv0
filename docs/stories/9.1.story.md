## Story

**Como** admin,
**Quero** escolher o modelo AI de cada agente entre diferentes providers,
**Para** usar o melhor modelo para cada tipo de análise (ex: Gemini para extração, Claude para síntese).

---

## Contexto

Atualmente `analyzeWithAgent()` em `src/lib/ai/agents/analyze.ts` está hardcoded para Google:

```typescript
const modelSlug = agent.model.replace('google/', '')
const { text, usage } = await generateText({
  model: google(modelSlug), // sempre google()
  ...
})
```

O campo `health_agents.model` já armazena formato `'google/gemini-2.5-flash'` com prefixo de provider. A factory function precisa parsear o prefixo e retornar o provider correto do Vercel AI SDK.

**Providers alvo:** Google (`@ai-sdk/google`), OpenAI (`@ai-sdk/openai`), Anthropic (`@ai-sdk/anthropic`)

---

## Acceptance Criteria

### AC1 — Factory function de provider

**Given** um string de modelo no formato `provider/model-name`
**When** `resolveModel('google/gemini-2.5-flash')` é chamado
**Then** retorna `google('gemini-2.5-flash')`

**Given** `resolveModel('openai/gpt-4o')`
**When** chamado
**Then** retorna `openai('gpt-4o')`

**Given** `resolveModel('anthropic/claude-sonnet-4-5')`
**When** chamado
**Then** retorna `anthropic('claude-sonnet-4-5')`

**Given** `resolveModel('formato-invalido')`
**When** formato não tem `/` ou provider desconhecido
**Then** retorna `google('gemini-2.5-flash')` como fallback
**And** loga warning no console

### AC2 — `analyzeWithAgent` usa factory

**Given** `analyzeWithAgent()` com `google()` hardcoded
**When** refatorado para usar `resolveModel(agent.model)`
**Then** o provider é determinado pelo campo `model` do agente no banco
**And** comportamento atual preservado (agentes existentes usam `google/gemini-2.5-flash`)

### AC3 — Orchestrators usam factory

**Given** `complete-analysis.ts` e `living-analysis.ts` usam `google()` diretamente para síntese
**When** refatorados
**Then** síntese também usa `resolveModel()` (ou usa um model configurável)

### AC4 — Admin form: seleção de provider/modelo

**Given** a tela de edição/criação de agente
**When** o admin edita o campo de modelo
**Then** existe dropdown com providers disponíveis: Google, OpenAI, Anthropic
**And** campo de texto para o slug do modelo (ex: `gemini-2.5-flash`, `gpt-4o`)
**And** preview do valor final: `google/gemini-2.5-flash`
**And** validação: formato `provider/model-name` obrigatório

### AC5 — Env vars por provider

**Given** providers diferentes requerem API keys diferentes
**When** um agente é configurado com `openai/gpt-4o`
**Then** o factory usa `process.env.OPENAI_API_KEY` (já existente como opcional)

**Given** um agente configurado com `anthropic/claude-sonnet-4-5`
**When** `ANTHROPIC_API_KEY` não está definida
**Then** `resolveModel` lança erro claro: "ANTHROPIC_API_KEY not configured"

### AC6 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Factory: resolve corretamente google, openai, anthropic
  - Factory: fallback para google em formato inválido
  - `analyzeWithAgent`: usa o provider do agente (mock verifica qual model foi passado)
  - Testes existentes continuam passando

## Dev Notes

### Factory function

```typescript
// src/lib/ai/core/resolve-model.ts
import { google } from '@ai-sdk/google'
import { openai } from '@ai-sdk/openai'
import { anthropic } from '@ai-sdk/anthropic'

const PROVIDERS = { google, openai, anthropic } as const
const DEFAULT_MODEL = 'google/gemini-2.5-flash'

export function resolveModel(modelString: string) {
  const slashIndex = modelString.indexOf('/')
  if (slashIndex === -1) {
    console.warn(`Invalid model format "${modelString}", using default`)
    return google('gemini-2.5-flash')
  }

  const providerName = modelString.slice(0, slashIndex) as keyof typeof PROVIDERS
  const modelSlug = modelString.slice(slashIndex + 1)

  const providerFn = PROVIDERS[providerName]
  if (!providerFn) {
    console.warn(`Unknown provider "${providerName}", using default`)
    return google('gemini-2.5-flash')
  }

  return providerFn(modelSlug)
}
```

### Packages existentes

### Orchestrator synthesis model

`complete-analysis.ts` e `living-analysis.ts` usam `google('gemini-2.5-flash')` direto para síntese. Pode ser um model fixo (configurável via env) ou um agente de síntese no banco. Approach mais simples: env `SYNTHESIS_MODEL` com default `google/gemini-2.5-flash`.
