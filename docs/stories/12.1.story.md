## Story

**Como** sistema,
**Quero** persistir sessões de chat com agentes e disponibilizar API de streaming,
**Para** que pacientes possam conversar com agentes especializados sobre suas análises.

---

## Contexto

O SAMI até agora gera análises one-shot (batch). O chat permite diálogo interativo com um agente, usando sua base de conhecimento segmentada (Story 8.1) + contexto da última análise. O streaming usa `streamText` do Vercel AI SDK.

---

## Acceptance Criteria

### AC1 — Schema `chat_sessions`

**Given** não existe tabela de chat
**When** migration é aplicada
**Then** existe tabela `chat_sessions` com:
  - `id` UUID PK
  - `userId` UUID FK → `users.id` ON DELETE CASCADE
  - `agentId` UUID FK → `health_agents.id`
  - `analysisId` UUID FK → `living_analyses.id` nullable (contexto de análise)
  - `title` TEXT (gerado automaticamente a partir da primeira mensagem)
  - `createdAt` TIMESTAMP DEFAULT NOW
  - `updatedAt` TIMESTAMP DEFAULT NOW

**And** índice em `(userId, updatedAt)` para listagem

### AC2 — Schema `chat_messages`

**Given** não existe tabela de mensagens
**When** migration é aplicada
**Then** existe tabela `chat_messages` com:
  - `id` UUID PK
  - `sessionId` UUID FK → `chat_sessions.id` ON DELETE CASCADE
  - `role` TEXT NOT NULL — `'user'` | `'assistant'`
  - `content` TEXT NOT NULL
  - `tokensUsed` INTEGER nullable
  - `createdAt` TIMESTAMP DEFAULT NOW

**And** índice em `sessionId` para carregamento de mensagens

### AC3 — API route de chat com streaming

**Given** o endpoint `POST /api/chat`
**When** um request autenticado é enviado com `{ sessionId, agentId, message }`
**Then:**
  1. Valida autenticação (`session.user.id`)
  2. Valida ownership da chat session
  3. Salva mensagem do usuário em `chat_messages`
  4. Carrega contexto: `systemPrompt` do agente + últimas 20 mensagens + RAG filtrada pelo agente (Story 8.1) + resumo da última análise
  5. Chama `streamText` com o contexto
  6. Retorna streaming response
  7. Após stream completo: salva mensagem do assistant + tokensUsed

### AC4 — Contexto do chat

**Given** um chat com agente "Nutrição"
**When** uma mensagem é enviada
**Then** o contexto inclui:
  - `system`: systemPrompt do agente + disclaimer educacional
  - `messages`: últimas 20 mensagens da sessão (sliding window)
  - Knowledge context: RAG filtrada pelo agente (`searchKnowledge` com `agentId`)
  - Análise context: resumo da última `living_analysis` do usuário (se existir)
  - NÃO inclui dados de outros pacientes
  - NÃO inclui conversas de outras sessões

### AC5 — Rate limiting

**Given** um paciente envia mensagens
**When** excede 30 mensagens na última hora
**Then** retorna `429 Too Many Requests` com mensagem: "Limite de mensagens atingido. Tente novamente em breve."

**Given** o paciente espera e envia após o período
**When** o limite é recalculado
**Then** pode enviar normalmente

### AC6 — Criação automática de sessão

**Given** o paciente inicia conversa com um agente sem sessão existente
**When** `POST /api/chat` é chamado com `sessionId: null`
**Then** cria nova `chat_sessions` com o agente
**And** `title` é gerado a partir dos primeiros 50 chars da primeira mensagem

**Given** o paciente já tem sessão ativa com o agente
**When** continua a conversa
**Then** usa a sessão existente (passada via `sessionId`)

### AC7 — Queries helper

**Given** as tabelas de chat
**When** queries são necessárias
**Then** existem:
  - `getChatSessionsByUser(userId): ChatSession[]` — ordenadas por `updatedAt` DESC
  - `getChatMessages(sessionId, limit=20): ChatMessage[]` — últimas N mensagens
  - `getSessionWithAgent(sessionId): ChatSession & { agent: HealthAgent }`

### AC8 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Schema: tabelas criadas com constraints corretas
  - API: autenticação, ownership validation, rate limiting (429)
  - Contexto: últimas 20 mensagens, RAG filtrada, análise incluída
  - Sessão: criação automática, reutilização existente
  - Streaming: `streamText` chamado com parâmetros corretos (mock)

## Dev Notes

### Streaming com Vercel AI SDK

```typescript
// src/app/api/chat/route.ts
import { streamText } from 'ai'

export async function POST(request: NextRequest) {
  // ... auth, validation, save user message ...

  const result = streamText({
    model: resolveModel(agent.model),
    system: buildChatSystemPrompt(agent, analysisContext, knowledgeContext),
    messages: chatHistory,
    temperature: Number(agent.temperature),
    onFinish: async ({ text, usage }) => {
      await saveChatMessage(sessionId, 'assistant', text, usage?.totalTokens)
    },
  })

  return result.toDataStreamResponse()
}
```

### Rate limiting — approach simples

```typescript
async function checkRateLimit(userId: string): Promise<boolean> {
  const oneHourAgo = new Date(Date.now() - 60 * 60 * 1000)
  const [{ count: msgCount }] = await db
    .select({ count: count() })
    .from(chatMessages)
    .innerJoin(chatSessions, eq(chatMessages.sessionId, chatSessions.id))
    .where(and(
      eq(chatSessions.userId, userId),
      eq(chatMessages.role, 'user'),
      gte(chatMessages.createdAt, oneHourAgo),
    ))
  return msgCount < 30
}
```

### Sliding window de mensagens

Carregar últimas 20 mensagens da sessão e converter para formato do AI SDK:

```typescript
const messages = await getChatMessages(sessionId, 20)
const chatHistory = messages.map(m => ({ role: m.role, content: m.content }))
```

### Knowledge context no chat

Usar a query do usuário como busca RAG:

```typescript
const ragChunks = await searchKnowledge(userMessage, 3, agent.id)
```

## Scope

**IN:** Schemas chat_sessions/chat_messages, API route streaming, rate limiting, context building, queries helper
**OUT:** Chat UI (Story 12.2), persistência de attachments/imagens no chat, moderação de conteúdo

## Complexity

**XL** (Extra Large) — 2 tabelas + API com streaming + rate limiting + context assembly + testes. ~6h estimado.

## Risks

- **MEDIUM:** Streaming com `onFinish` callback pode ter edge cases de race condition ao salvar. Mitigação: usar `await` no callback.
- **LOW:** Rate limiting por count de mensagens pode ser burlado com refresh. Aceitável para v1.1 — rate limiting mais robusto em futuro.
- **LOW:** Consultar docs atualizados do AI SDK para API de streaming (v6 mudou nomes de métodos).
