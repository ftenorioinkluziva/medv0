# Story 6.3 — Gestão da Knowledge Base (Admin)

**Status:** Ready for Review
**Linear:** BLA-54
**Branch:** `ftenorioemail/bla-54-story-63-gestao-knowledge-base`
**Cycle:** 1 (30/03–06/04/2026)

## Story

**Como** admin,
**Quero** visualizar, buscar e remover artigos indexados no RAG,
**Para** manter a Knowledge Base organizada e garantir a qualidade do contexto disponível para os agentes de IA.

---

## Acceptance Criteria

### AC1 — Listagem de artigos

- Rota `/admin/knowledge` exibe tabela com: título, categoria, `usage_count`, `is_verified`, data de criação
- Tabela paginada ou com scroll, ordenada por `createdAt` desc

### AC2 — Busca por título/categoria

- Campo de busca filtra artigos por título (ilike) ou categoria (ilike) em tempo real (debounce ou submit)
- Estado de busca vazia exibe mensagem "Nenhum artigo encontrado"

### AC3 — Remoção com cascade de embeddings

- Botão "Remover" em cada linha abre `AlertDialog` de confirmação
- Ao confirmar, deleta o artigo + todos os embeddings associados em transação (ON DELETE CASCADE já configurado no schema)
- Após remoção: revalidar `/admin/knowledge`, exibir toast de sucesso
- Erro na remoção exibe toast de erro

### AC4 — Navegação no admin nav

- Link "Knowledge Base" no `AdminNav` (já existe: `href: '/admin/knowledge'`) navega para a rota correta

---

## Scope

### IN
- Listagem de artigos (`knowledge_base`)
- Busca por título e categoria
- Remoção de artigo (com cascade para `knowledge_embeddings`)
- Server Action para delete
- Query Drizzle para listagem com filtros
- Toast feedback (Sonner)

### OUT
- Criação/edição de artigos via UI (já coberta pelo endpoint auto-upload)
- Upload de arquivos
- Verificação manual (`is_verified` apenas exibido, não editável aqui)
- Paginação complexa (cursor/infinite) — tabela simples é suficiente

## Risks

- `is_verified` é `text` no schema (não boolean) — exibir valor raw ou mapear para badge colorido
- Busca com ILIKE pode ser lenta em tabelas grandes — aceitável para MVP (knowledge base é pequena)
- Cascade de embeddings já está no schema via `onDelete: 'cascade'` — delete simples é suficiente

### API / Server Actions
Não há endpoints REST novos — tudo via Server Actions (`'use server'`), seguindo o padrão de `src/app/admin/agents/_actions/agents.ts`.

**Delete action:**
```typescript
// cascade via FK — basta deletar o artigo
await db.delete(knowledgeBase).where(eq(knowledgeBase.id, id))
revalidatePath('/admin/knowledge')
```

**List query com filtro:**
```typescript
import { ilike, or, asc, desc } from 'drizzle-orm'

// sem filtro
db.select().from(knowledgeBase).orderBy(desc(knowledgeBase.createdAt))

// com filtro de busca
db.select().from(knowledgeBase)
  .where(or(
    ilike(knowledgeBase.title, `%${query}%`),
    ilike(knowledgeBase.category, `%${query}%`)
  ))
  .orderBy(desc(knowledgeBase.createdAt))
```
