## Story

**Como** paciente,
**Quero** completar meu perfil com dados de estilo de vida e cronobiologia,
**Para** que as análises considerem meu contexto de vida completo.

---

## Acceptance Criteria

### AC1 — Schema avançado (adições à `medical_profiles`)

Sono: `sleep_hours`, `sleep_quality` (1-10), `sleep_issues`, `time_in_bed`, `sleep_regularity`
Estilo de vida: `daily_water_intake`, `stress_level` (1-10), `stress_management`, `smoking_status`, `smoking_details`, `alcohol_consumption`, `supplementation` (text[]), `current_diet`
Atividade física: `exercise_activities` (jsonb), `exercise_types` (text[]), `exercise_frequency`, `exercise_duration`, `exercise_intensity`, `physical_limitations`
Cronobiologia: `first_sunlight_exposure_time`, `last_meal_time`, `artificial_light_exposure_start`, `artificial_light_exposure_end`, `artificial_light_exposure_time`

### AC2 — Server Action atualizado

- `upsertMedicalProfile()` aceita campos avançados (Zod expandido)
- Todos os campos avançados opcionais no Zod
- Upsert parcial (não apaga campos existentes)

### AC3 — Formulário avançado

- Seção "Dados Avançados" em `/app/profile`
- Sub-seções: Sono, Atividade Física, Hábitos, Cronobiologia
- Lista dinâmica de atividades físicas (add/remove)
- Inputs HH:MM para cronobiologia

### AC4 — Todos opcionais

- Form submete com todos os campos avançados vazios
- Análise funciona sem dados avançados
