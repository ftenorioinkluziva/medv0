## Story

**Como** admin,
**Quero** criar, visualizar, editar e excluir agentes de IA via painel administrativo,
**Para** configurar o pipeline de análise médica sem necessidade de deploy ou alterações em código.

---

## Acceptance Criteria

### AC1 — Listagem de agentes

- Rota `/admin/agents` exibe tabela com nome, especialidade, role, modelo e status (ativo/inativo)
- Botão "Novo Agente" navega para `/admin/agents/new`

### AC2 — Criar agente

- Formulário em `/admin/agents/new` com campos: nome, especialidade, descrição, systemPrompt, role, modelo, temperature, maxTokens, sortOrder (quando role=foundation), isActive
- `systemPrompt` exige mínimo 50 caracteres (validação server-side)
- Ao salvar com sucesso, redireciona para `/admin/agents`

### AC3 — Editar agente

- Rota `/admin/agents/[id]/edit` pré-popula formulário com dados do agente
- Dados persistem ao salvar; redireciona para `/admin/agents`

### AC4 — Toggle ativo/inativo

- Switch na listagem alterna status do agente
- Confirmação via AlertDialog antes de desativar
- Guard: não permite desativar se for o único foundation agent ativo
  - Mensagem: "Pelo menos 1 agente foundation deve estar ativo"

### AC5 — Excluir agente

- Botão "Excluir" abre AlertDialog de confirmação
- Guard: não permite excluir se for o único foundation agent ativo
  - Mensagem: "Pelo menos 1 agente foundation deve estar ativo. Não é possível excluir o único foundation ativo."
- Exclusão permanente após confirmação

### Cuidados técnicos

- **SAMI-DEBT-001**: `updatedAt` NÃO é auto-atualizado pelo DB em UPDATEs. Sempre incluir `.set({ updatedAt: new Date() })` em toda operação de update.
- **temperature como string**: Drizzle retorna `numeric` como `string`. Usar `parseFloat(String(agent.temperature))` antes de usar como número.
- **Zod v4**: Usar `.issues[0]?.message` (não `.errors`).
- **Base UI**: Projeto usa `@base-ui/react`, não Radix UI. Sem `asChild` — usar prop `render` para links: `<Button render={<Link href="..." />}>`. AlertDialog controlado via `useState` + `open/onOpenChange`.
- **Next.js 16**: `params` é Promise — usar `const { id } = await params`.

#### Observations

- **MNT-001 (medium):** `updateAgentAction` guards only role changes, not `isActive` changes. Last active foundation agent can be deactivated via the edit form, bypassing the `toggleAgentAction` guard. Recommended fix: add guard for `agent.isActive === true && parsed.data.isActive === false && agent.analysisRole === 'foundation'`.
- **TEST-001 (medium):** `updateAgentAction` has zero unit test coverage. Three branches untested (not found, role-change guard, happy path).
- **SEC-001 (low):** Server Actions lack own `auth()` check. Middleware provides adequate first-layer protection. Defence-in-depth improvement deferred to tech debt.
