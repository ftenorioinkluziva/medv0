## Story

**Como** admin,
**Quero** tabelas paginadas e código limpo sem vestígios de roles não implementadas,
**Para** que o painel admin escale com o crescimento de usuários e o código reflita apenas o que existe.

---

## Contexto

**Paginação:** `getAllUsersForAdmin()` (`src/lib/db/queries/users.ts`), `getAllArticlesForAdmin()` (`src/lib/db/queries/knowledge.ts`) e `getAllAgentsForAdmin()` (`src/lib/db/queries/health-agents.ts`) retornam TODOS os registros sem limit/offset.

**Role doctor:** `src/proxy.ts:4` define `type Role = 'admin' | 'patient' | 'doctor'` mas o enum no DB é apenas `['patient', 'admin']`. Role `doctor` nunca foi implementada.

**Orchestrator duplication:** `complete-analysis.ts` (~300 linhas) e `living-analysis.ts` (~350 linhas) compartilham: `readTimeoutMs()`, construção de `medicalProfileContext`, loop de foundation agents com AbortController, `Promise.allSettled` de specialized, chamada de síntese. ~250 linhas duplicadas.

---

## Acceptance Criteria

### AC1 — Paginação em queries admin

**Given** `getAllUsersForAdmin()` retorna todos os registros
**When** assinatura é atualizada para `getAllUsersForAdmin(limit?: number, offset?: number)`
**Then** com `limit=20, offset=0` retorna no máximo 20 registros
**And** retorna também `total: number` para cálculo de páginas
**And** sem parâmetros mantém comportamento atual (default `limit=50, offset=0`)

**Given** as mesmas mudanças em `getAllArticlesForAdmin()` e `getAllAgentsForAdmin()`
**When** chamadas com limit/offset
**Then** paginam corretamente com total count

### AC2 — Tabelas admin com paginação via query params

**Given** as páginas admin de users, knowledge e agents
**When** acessadas com `?page=2&size=20`
**Then** mostram a página correspondente com navegação (anterior/próximo)
**And** default: `page=1, size=20`

### AC3 — Remover role `doctor` do proxy

**Given** `proxy.ts` linha 4: `type Role = 'admin' | 'patient' | 'doctor'`
**When** a role `doctor` é removida
**Then** `type Role = 'admin' | 'patient'`
**And** nenhuma referência a `doctor` existe no codebase (grep confirma)

### AC4 — Deduplicar orchestrators

**Given** `complete-analysis.ts` e `living-analysis.ts` com ~250 linhas duplicadas
**When** pipeline compartilhado é extraído
**Then** existe `src/lib/ai/orchestrator/pipeline.ts` com:
  - `readTimeoutMs(envName, fallbackMs)`
  - `buildMedicalProfileContext(userId)`
  - `runFoundationPhase(agents, context, signal)`
  - `runSpecializedPhase(agents, context, signal)`
  - `runSynthesisPhase(agentOutputs, context, signal, synthesisPrompt)`

**Given** o refactor é feito
**When** `complete-analysis.ts` e `living-analysis.ts` usam o pipeline extraído
**Then** ambos mantêm comportamento idêntico ao atual (mesmas timeouts, mesma lógica)
**And** testes existentes continuam passando sem alteração de assertions

### AC5 — Testes

**Given** as mudanças de paginação e pipeline
**When** testes são executados
**Then:**
  - Queries admin: paginação com limit/offset retorna subset correto + total count
  - Pipeline: testes existentes de `runCompleteAnalysis` e `runLivingAnalysis` passam (comportamento preservado)

## Dev Notes

### Padrão de paginação proposto

```typescript
// src/lib/db/queries/users.ts
export interface PaginatedResult<T> {
  data: T[]
  total: number
}

export async function getAllUsersForAdmin(
  limit: number = 50,
  offset: number = 0,
): Promise<PaginatedResult<UserForAdmin>> {
  const [rows, totalResult] = await Promise.all([
    db.select({ ... }).from(users).limit(limit).offset(offset),
    db.select({ count: count() }).from(users),
  ])
  return { data: rows, total: totalResult[0]?.count ?? 0 }
}
```

### Pipeline extraction — funções compartilhadas

Ambos orchestrators fazem:
1. `readTimeoutMs(envName, fallbackMs)` — idêntico
2. Load snapshot + profile → build context strings — quase idêntico (living-analysis adiciona `previousAnalysisContext`)
3. Foundation loop com AbortController — idêntico exceto prompt
4. Specialized `Promise.allSettled` — idêntico
5. Synthesis com generateText — idêntico exceto prompt

O pipeline deve receber o prompt como parâmetro, não hardcodar.

### Proxy type — change mínimo

```diff
- type Role = 'admin' | 'patient' | 'doctor'
+ type Role = 'admin' | 'patient'
```
