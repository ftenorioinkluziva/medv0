## Story

**Como** paciente,
**Quero** enviar um exame médico (PDF ou imagem) pelo meu celular,
**Para** que o sistema possa processar e extrair os dados.

---

## Contexto

Ponto de entrada do fluxo de ingestão. O arquivo nunca persiste no servidor — é processado em memória e descartado. A interface deve ser mobile-first, com câmera nativa no iOS/Android. O processamento pode levar até 30s (NFR4).

---

## Acceptance Criteria

### AC1 — Interface de upload mobile-first
* Tela `/app/upload` (ou modal a partir do dashboard)
* Dois modos de seleção:
  1. **Câmera** — `<input type="file" accept="image/*" capture="environment">` → abre câmera nativa no mobile
  2. **Galeria/Arquivos** — `<input type="file" accept="image/jpeg,image/png,application/pdf">`
* Shadcn UI: Card, Button, Progress

### AC2 — Validação client-side antes do envio
* Tipos aceitos: `image/jpeg`, `image/png`, `application/pdf`
* Tamanho máximo: 20MB — verificação no cliente antes de submeter (erro imediato sem round-trip)
* Mensagem de erro clara para tipo inválido ou arquivo muito grande
* Preview do arquivo: thumbnail para imagens, nome + ícone para PDFs

### AC3 — Envio e feedback de progresso
* Arquivo enviado via `multipart/form-data` para Route Handler `POST /api/documents/upload`
* **Route Handler** (não Server Action) — necessário para lidar com streaming multipart e timeouts longos (30s)
* Barra de progresso com estados: Preparando → Enviando → Extraindo → Salvando → Concluído
* Botão de cancelamento durante envio
* Timeout visual ao usuário se >30s (NFR4)

### AC4 — Arquivo nunca persiste no servidor
* Buffer in-memory apenas (`Buffer.from(await file.arrayBuffer())`)
* Nenhuma escrita em disco ou storage (Blob, S3, etc.)
* Arquivo destruído imediatamente após extração e persistência do snapshot

### AC5 — Feedback de conclusão
* Sucesso: toast com "Exame processado!" + CTA para ver análise ou dashboard
* Erro do servidor: mensagem amigável sem expor detalhes técnicos
* Retry automático não implementado — usuário pode tentar novamente manualmente
