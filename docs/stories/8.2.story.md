## Story

**Como** admin,
**Quero** associar artigos a agentes específicos na interface de administração,
**Para** controlar quais artigos cada agente consulta e segmentar a base de conhecimento.

---

## Contexto

Story 8.1 cria a tabela `agent_knowledge` e o campo `isGlobal`. Esta story constrói a UI admin para gerenciar essas associações. Atualmente `src/app/admin/agents/` lista agentes e `src/app/admin/knowledge/` lista artigos — nenhum mostra a relação entre eles.

---

## Acceptance Criteria

### AC1 — Seção "Base de Conhecimento" na edição do agente

**Given** o admin está na tela de edição de um agente (`/admin/agents/[id]/edit`)
**When** a página carrega
**Then** existe seção "Base de Conhecimento" abaixo dos campos do agente
**And** lista os artigos atualmente associados ao agente (título + categoria)
**And** mostra count de artigos associados

### AC2 — Busca e associação de artigos

**Given** a seção "Base de Conhecimento" na edição do agente
**When** o admin digita no campo de busca
**Then** resultados filtram artigos por título ou categoria (busca server-side)
**And** artigos já associados aparecem com indicador visual (check)
**And** clicar em um artigo não-associado o associa imediatamente
**And** clicar em um artigo associado o desassocia

### AC3 — Toggle `isGlobal` em artigos

**Given** a listagem de knowledge base (`/admin/knowledge`)
**When** o admin vê um artigo
**Then** existe toggle "Global" que controla `isGlobal`
**And** artigos globais são acessíveis por todos os agentes independentemente de associação
**And** toggle reflete estado atual e persiste alteração via Server Action

### AC4 — Indicador de agentes na listagem de artigos

**Given** a listagem de knowledge base (`/admin/knowledge`)
**When** o admin vê um artigo
**Then** existe coluna/badge mostrando quais agentes usam aquele artigo
**And** artigos globais mostram badge "Global" em destaque
**And** artigos sem associação e não-globais mostram indicador de "sem agente"

### AC5 — Bulk actions

**Given** a seção "Base de Conhecimento" na edição do agente
**When** o admin seleciona múltiplos artigos via checkbox
**Then** pode associar todos de uma vez ("Adicionar selecionados")
**And** pode desassociar todos de uma vez ("Remover selecionados")

### AC6 — Server Actions com auth

**Given** as operações de associação/desassociação/toggle global
**When** executadas
**Then** todas as Server Actions verificam `requireAdmin()` (Story 7.2)
**And** retornam erro se não autorizado

## Dev Notes

### Server Actions

```typescript
// src/lib/actions/admin-knowledge.ts
'use server'
import { requireAdmin } from '@/lib/auth/require-admin'
import { associateArticleToAgent, disassociateArticleFromAgent } from '@/lib/db/queries/knowledge'

export async function associateArticlesAction(agentId: string, articleIds: string[]) {
  await requireAdmin()
  for (const articleId of articleIds) {
    await associateArticleToAgent(agentId, articleId)
  }
}
```

## Scope
