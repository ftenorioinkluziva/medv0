## Story

**Como** admin,
**Quero** que todas as operações admin verifiquem minha role independentemente do middleware,
**Para** que nenhuma Server Action ou API route admin seja acessível por pacientes mesmo em caso de bypass de middleware.

---

## Contexto

Atualmente a proteção admin depende exclusivamente do `layout.tsx` (Server Component que verifica `session.user.role !== 'admin'`). Não existem Server Actions no admin — as pages fazem queries diretas (`getAllAgentsForAdmin()`, `getAllUsersForAdmin()`, etc.). O endpoint `POST /api/admin/knowledge/auto-upload` não tem autenticação alguma. O middleware (`proxy.ts`) protege rotas `/admin/*` mas não cobre API routes.

**Superfície de ataque:**
- `auto-upload` acessível por qualquer pessoa na internet
- Se uma Server Action for adicionada no futuro sem auth check, ficará exposta
- API routes admin não verificam role

---

## Acceptance Criteria

### AC1 — Server Actions admin com auth guard

**Given** operações admin (toggle agent, delete article, deactivate user) existem como queries diretas em Server Components
**When** são extraídas para Server Actions (`"use server"`)
**Then** cada Server Action:
  1. Chama `auth()` no início
  2. Verifica `session?.user?.role === 'admin'`
  3. Retorna `{ error: 'Unauthorized' }` se role inválida
  4. Executa a operação apenas se autorizado

### AC2 — Endpoint auto-upload protegido

**Given** `POST /api/admin/knowledge/auto-upload` sem autenticação
**When** um request chega sem credenciais
**Then** retorna `401 Unauthorized`

**Given** um request com header `x-api-key` válido (env `KNOWLEDGE_UPLOAD_API_KEY`)
**When** o valor do header corresponde à env var
**Then** o request é processado normalmente

**Given** um request com session de admin válida (cookie NextAuth)
**When** `session.user.role === 'admin'`
**Then** o request é processado normalmente (fallback para admin logado)

**Given** o `GET` (check duplicata) do mesmo endpoint
**When** acessado sem auth
**Then** também retorna `401` (mesma proteção)

### AC3 — Helper reutilizável de auth admin

**Given** a necessidade de verificar admin role em múltiplos locais
**When** `requireAdmin()` é chamado
**Then** retorna a session se `role === 'admin'`
**And** lança/retorna erro padronizado se não autorizado

```typescript
// src/lib/auth/require-admin.ts
export async function requireAdmin(): Promise<Session>
```

### AC4 — Testes de acesso indevido

**Given** um usuário com role `patient`
**When** tenta acessar cada Server Action admin
**Then** recebe `{ error: 'Unauthorized' }` em todos os casos

**Given** um request sem session
**When** tenta acessar `POST /api/admin/knowledge/auto-upload`
**Then** recebe `401`

**Given** um request com `x-api-key` inválido
**When** tenta acessar `POST /api/admin/knowledge/auto-upload`
**Then** recebe `401`

## Dev Notes

### Padrão de auth existente

```typescript
// src/app/admin/layout.tsx — padrão atual (Server Component)
import { auth } from '@/lib/auth/config'
const session = await auth()
if (!session || session.user.role !== 'admin') {
  redirect('/auth/login')
}
```

### Helper proposto

```typescript
// src/lib/auth/require-admin.ts
import { auth } from '@/lib/auth/config'

export async function requireAdmin() {
  const session = await auth()
  if (!session?.user?.id || session.user.role !== 'admin') {
    throw new Error('Unauthorized')
  }
  return session
}
```

### Auto-upload — lógica de auth dual

```typescript
// src/app/api/admin/knowledge/auto-upload/route.ts
function isAuthorized(request: NextRequest, session: Session | null): boolean {
  const apiKey = request.headers.get('x-api-key')
  if (apiKey && apiKey === process.env.KNOWLEDGE_UPLOAD_API_KEY) return true
  if (session?.user?.role === 'admin') return true
  return false
}
```

### Env var necessária

```bash
KNOWLEDGE_UPLOAD_API_KEY="..."  # Gerar com openssl rand -hex 32
```

### Admin pages que fazem queries diretas (precisam de Server Actions)

- `src/app/admin/agents/page.tsx` → `getAllAgentsForAdmin()`
- `src/app/admin/knowledge/page.tsx` → `getAllArticlesForAdmin()`
- `src/app/admin/users/page.tsx` → `getAllUsersForAdmin()`

As queries de **leitura** em Server Components já são protegidas pelo layout.tsx. Server Actions são necessárias para **mutações** (toggle, delete, deactivate).

## Scope

**IN:**
- Auth guard em Server Actions de mutação admin
- Auth em auto-upload (API key + session)
- Helper reutilizável

**OUT:**
- Redesign completo das páginas admin (apenas extração de mutações)
- Rate limiting no auto-upload (futuro)
- Proteção de API routes de leitura (já protegidas pelo layout)
