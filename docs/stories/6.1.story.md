## Story

**Como** admin,
**Quero** acessar um painel separado com autenticação por role,
**Para** gerenciar agentes, knowledge base e usuários sem interferência do fluxo de pacientes.

---

## Contexto

Pré-requisito de todo o Épico 6. Protege `/admin/*` para role `admin`, redireciona pacientes para `/app`, e redireciona admins de `/` direto para `/admin`. Admin criado exclusivamente via seed (não há cadastro público).

---

## Acceptance Criteria

### AC1 — Rotas `/admin/*` protegidas

- Acesso sem sessão → redirect para `/auth/login?callbackUrl=/admin`
- Sessão com `role !== 'admin'` → redirect para `/`
- Sessão com `role === 'admin'` → acesso permitido

### AC2 — Redirecionamento de admin em `/app/*`

- Admin autenticado acessando `/app/*` → redirect para `/admin`
- Admin autenticado acessando `/` → redirect para `/admin`

### AC3 — Layout separado `/admin`

- `app/admin/layout.tsx` com Server Component
- `AdminNav` com navegação lateral (desktop) e drawer (mobile)
- Exibe nome/email do admin logado

### AC4 — Seed de admin

- `lib/db/seed/admin.ts` — seed idempotente via `ON CONFLICT DO NOTHING`
- Lê `ADMIN_EMAIL` e `ADMIN_PASSWORD` de env vars
- Skip com warning se `ADMIN_PASSWORD` não definido
- Integrado ao `lib/db/seed.ts` principal

### AC5 — Sessão expõe `name`

- `lib/auth/config.ts` JWT callback propaga `name` do usuário
- AdminNav usa `session.user.name ?? session.user.email`
