## Story

**Como** paciente,
**Quero** ver planos de treino renderizados como cards interativos,
**Para** consumir o plano de forma visual e prática em vez de texto corrido.

---

## Contexto

Agentes com `outputType: 'structured'` geram JSON tipado persistido em `analyses.content`. Atualmente o relatório renderiza tudo como Markdown. Agentes estruturados precisam de componentes UI dedicados que renderizam o JSON como cards interativos.

---

## Acceptance Criteria

### AC1 — Detector de output estruturado

**Given** o relatório da análise exibe outputs de cada agente
**When** um agente tem `outputType: 'structured'` e output JSON válido
**Then** renderiza componente especializado em vez de Markdown
**And** agentes com `outputType: 'text'` continuam renderizando Markdown normalmente

### AC2 — Workout plan component

**Given** output JSON de um agente de workout plan (schema AC2 da Story 10.1)
**When** renderizado
**Then** exibe:
  - Overview em texto no topo
  - Cards por dia da semana com: tipo de treino, duração
  - Lista de exercícios em cada card: nome, séries, repetições, notas
  - Seções de warmup e cooldown colapsáveis
  - Dias de descanso indicados
  - Meta semanal destacada
  - Tips de progressão em lista

### AC3 — Generic JSON renderer (fallback)

**Given** um agente com `outputType: 'structured'` mas sem componente dedicado registrado
**When** output JSON é renderizado
**Then** usa renderer genérico que exibe o JSON em formato legível:
  - Objetos como seções com heading
  - Arrays como listas
  - Valores primitivos em texto formatado
**And** NÃO exibe JSON bruto — sempre formatado para leitura humana

### AC4 — Posicionamento no relatório

**Given** a análise completa tem agentes text e structured
**When** o relatório é renderizado
**Then** outputs estruturados aparecem abaixo da síntese integrativa
**And** cada output estruturado tem heading com nome do agente e specialty
**And** disclaimer educacional aparece abaixo de cada plano

### AC5 — Responsividade mobile (390px)

**Given** o workout component em viewport 390px
**When** renderizado
**Then** cards de dias da semana empilham verticalmente
**And** lista de exercícios usa layout compacto (nome + sets/reps em uma linha)
**And** nenhum horizontal overflow

### AC6 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Detector: text → Markdown, structured → componente especializado
  - Workout component: renderiza todos os campos do schema
  - Generic renderer: formata JSON arbitrário sem mostrar raw JSON
  - Playwright: layout correto em 390px

## Dev Notes

### Component registry

```typescript
// src/components/structured-outputs/registry.ts
const STRUCTURED_COMPONENTS: Record<string, React.ComponentType<{ data: unknown }>> = {
  'Plano de Exercícios': WorkoutPlanView,
  // futuras: 'Plano Alimentar': NutritionPlanView, etc.
}

export function getStructuredComponent(agentName: string) {
  return STRUCTURED_COMPONENTS[agentName] ?? GenericStructuredView
}
```

### Detector no relatório

O relatório acessa `analyses[]` da `complete_analysis`/`living_analysis`. Cada `analysis` tem:
- `agentName`: nome do agente
- `content`: texto ou JSON stringified
- `analysisRole`: 'foundation' | 'specialized'

Para agentes structured, parsear `content` como JSON e passar ao componente.

### Onde renderizar

Verificar onde o relatório é renderizado — provavelmente `src/app/app/analyses/[id]/page.tsx`. Os outputs individuais dos agentes podem estar no `analysisData` JSONB da `livingAnalysisVersions` ou `completeAnalyses`.
