## Story

**Como** paciente,
**Quero** conversar com um agente especializado para tirar dúvidas sobre minha saúde,
**Para** entender melhor minha análise e receber orientações personalizadas de forma interativa.

---

## Contexto

Story 12.1 cria o backend de chat com streaming. Esta story constrói a interface mobile-first de chat com seleção de agente, streaming de mensagens e histórico de sessões.

---

## Acceptance Criteria

### AC1 — Tela de seleção de agente

**Given** o paciente acessa a tela de chat (`/app/chat`)
**When** a página carrega
**Then** exibe lista de agentes disponíveis (ativos) com:
  - Nome do agente
  - Specialty
  - Descrição curta
**And** cada agente é um card clicável

### AC2 — Início de conversa

**Given** o paciente seleciona um agente
**When** clica no card
**Then:**
  - Se existe sessão anterior com esse agente: abre a última sessão com histórico
  - Se não existe: cria nova sessão e abre chat vazio
**And** header mostra: "{agente.name} — {agente.specialty}"
**And** disclaimer no topo: "Esta conversa é para fins educacionais e NÃO substitui consulta médica profissional."

### AC3 — Interface de mensagens com streaming

**Given** o paciente está em uma sessão de chat
**When** digita e envia uma mensagem
**Then:**
  - Mensagem do usuário aparece imediatamente (bubble à direita)
  - Indicator de "digitando..." aparece
  - Resposta do agente é exibida com streaming (texto aparecendo progressivamente, bubble à esquerda)
  - Campo de input é limpo após envio
  - Scroll automático para a última mensagem

### AC4 — Integração com Vercel AI SDK `useChat`

**Given** a UI de chat
**When** implementada
**Then** usa `useChat` hook do `@ai-sdk/react` (ou equivalente atual) para:
  - Gerenciar estado de mensagens
  - Enviar mensagens ao `POST /api/chat`
  - Receber streaming response
  - Exibir estado de loading

### AC5 — Histórico de sessões

**Given** o paciente tem múltiplas sessões de chat
**When** acessa `/app/chat`
**Then** abaixo dos agentes existe seção "Conversas anteriores" com:
  - Lista de sessões ordenada por `updatedAt` DESC
  - Cada item: título da sessão + nome do agente + data
  - Clicar abre a sessão com histórico de mensagens carregado

### AC6 — Indicador de rate limit

**Given** o paciente atingiu o limite de 30 mensagens/hora
**When** tenta enviar nova mensagem
**Then** input é desabilitado com mensagem: "Limite de mensagens atingido. Tente novamente em breve."
**And** não faz request ao servidor

### AC7 — Mobile-first (390px)

**Given** a tela de chat em viewport 390px
**When** visualizada
**Then:**
  - Mensagens ocupam largura total com margem mínima
  - Input fixo na parte inferior (acima da bottom nav)
  - Teclado virtual não esconde o input
  - Bubbles de mensagem com max-width 85%
  - Scroll suave no container de mensagens

### AC8 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Seleção de agente: lista agentes ativos
  - Sessão: cria nova ou retoma existente
  - Streaming: mensagens aparecem progressivamente
  - Histórico: sessões anteriores listadas e acessíveis
  - Rate limit: input desabilitado ao atingir limite
  - Playwright: layout correto em 390px, envio de mensagem funciona

## Dev Notes

### `useChat` hook

Verificar documentação atual do Vercel AI SDK para o hook de chat correto. O projeto usa `@ai-sdk/react` ou `ai/react`. A assinatura típica:

```typescript
import { useChat } from '@ai-sdk/react'

const { messages, input, handleInputChange, handleSubmit, isLoading } = useChat({
  api: '/api/chat',
  body: { sessionId, agentId },
})
```

Consultar docs atualizados antes de implementar — API pode ter mudado.

## Scope

**IN:** Tela de seleção de agente, interface de chat com streaming, histórico de sessões, rate limit UI, mobile layout
**OUT:** Chat com múltiplos agentes simultâneos, voice messages, file sharing no chat, notificações push

## Complexity

**XL** (Extra Large) — 2 pages novas + streaming UI + histórico + responsividade + Playwright. ~6h estimado.

## Risks

- **MEDIUM:** API do `useChat` hook mudou no AI SDK v6 — @dev deve consultar docs atualizados (`sendMessage` em vez de `handleSubmit`, `status` em vez de `isLoading`).
- **LOW:** Teclado virtual em mobile pode cobrir o input. Mitigação: `position: sticky` + `bottom: 0` + viewport meta tag.
