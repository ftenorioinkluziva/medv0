## Story

**Como** sistema,
**Quero** ter o schema da knowledge base e busca por similaridade vetorial,
**Para** que o motor de análise possa consultar artigos médicos verificados via RAG.

---

## Contexto

Base da camada RAG do SAMI. Artigos médicos são armazenados com embeddings vetoriais (768 dimensões, Google text-embedding-004) e consultados por cosine similarity via pgvector. A busca é usada pelos agentes de IA em E4 para fundamentar análises com fontes verificadas.

---

## Acceptance Criteria

### AC1 — Schema `knowledge_base`

Colunas:
- `id` (UUID, PK)
- `title` (text, not null)
- `content` (text, not null)
- `summary` (text, nullable)
- `source` (text, nullable)
- `author` (text, nullable)
- `published_date` (date, nullable)
- `category` (text, nullable)
- `subcategory` (text, nullable)
- `tags` (text[], nullable)
- `language` (text, default `pt-BR`)
- `is_verified` (text, default `unverified`)
- `usage_count` (integer, default 0)
- `last_analyzed_at` (timestamp, nullable)
- `analysis_version` (text, nullable)
- `created_at`, `updated_at` (timestamps)

### AC2 — Schema `knowledge_embeddings`

Colunas:
- `id` (UUID, PK)
- `article_id` (UUID, FK → knowledge_base, cascade delete)
- `chunk_index` (integer, not null)
- `content` (text, not null)
- `embedding` (vector(768), not null)
- `created_at` (timestamp)

Índice HNSW: `vector_cosine_ops` (m=16, ef_construction=64)

### AC3 — `searchKnowledge(query, topK)`

- Gera embedding da query via `embed()` com `text-embedding-004`
- Busca por `cosineDistance` com `desc(similarity)`
- Retorna `KnowledgeChunk[]`: articleId, chunkIndex, content, score, article metadata
- Default `topK = 5`

### AC4 — `incrementUsageCount(articleId)`

- UPDATE atômico: `usage_count + 1` + `last_analyzed_at = now()`
- Usado pelos agentes após cada consulta RAG

### AC5 — Migration aplicada

- Migration `0004` com CREATE TABLE knowledge_base + knowledge_embeddings + índice HNSW
- Aplicada sem erros no Neon Postgres
