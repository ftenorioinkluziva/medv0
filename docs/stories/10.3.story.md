## Story

**Como** paciente,
**Quero** planos alimentar e de suplementação personalizados,
**Para** receber orientação prática de nutrição e suplementos baseada nos meus exames.

---

## Contexto

Após Story 10.1 (workout) e 10.2 (UI), esta story aplica o mesmo padrão para dois novos agentes estruturados: "Plano Alimentar" e "Plano de Suplementação". Ambos são specialized agents que recebem foundation context + Medical Profile.

---

## Acceptance Criteria

### AC1 — Health agent "Plano Alimentar" no seed

**Given** o seed de agentes
**When** executado
**Then** existe agente "Plano Alimentar" com:
  - `analysisRole`: `'specialized'`
  - `outputType`: `'structured'`
  - `outputSchema`: schema de nutrition plan (AC3)
  - `systemPrompt`: prompt que considera dieta atual, alergias, condições, objetivos

### AC2 — Health agent "Plano de Suplementação" no seed

**Given** o seed de agentes
**When** executado
**Then** existe agente "Plano de Suplementação" com:
  - `analysisRole`: `'specialized'`
  - `outputType`: `'structured'`
  - `outputSchema`: schema de supplement plan (AC4)
  - `systemPrompt`: prompt que considera medicamentos, condições, interações

### AC3 — Output schema do nutrition plan

**Given** o agente "Plano Alimentar"
**When** `outputSchema` é definido
**Then** inclui:

```json
{
  "type": "object",
  "properties": {
    "overview": { "type": "string" },
    "weeklyGoal": { "type": "string" },
    "meals": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "meal": { "type": "string" },
          "time": { "type": "string" },
          "foods": { "type": "array", "items": { "type": "string" } },
          "calories": { "type": "number" },
          "macros": {
            "type": "object",
            "properties": {
              "protein": { "type": "string" },
              "carbs": { "type": "string" },
              "fat": { "type": "string" }
            }
          }
        },
        "required": ["meal", "foods"]
      }
    },
    "restrictions": { "type": "array", "items": { "type": "string" } },
    "hydration": { "type": "string" },
    "notes": { "type": "string" }
  },
  "required": ["overview", "meals", "weeklyGoal"]
}
```

### AC4 — Output schema do supplement plan

**Given** o agente "Plano de Suplementação"
**When** `outputSchema` é definido
**Then** inclui:

```json
{
  "type": "object",
  "properties": {
    "overview": { "type": "string" },
    "supplements": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "name": { "type": "string" },
          "dosage": { "type": "string" },
          "timing": { "type": "string" },
          "purpose": { "type": "string" },
          "interactions": { "type": "array", "items": { "type": "string" } }
        },
        "required": ["name", "dosage", "timing", "purpose"]
      }
    },
    "reviewDate": { "type": "string" },
    "warnings": { "type": "array", "items": { "type": "string" } },
    "notes": { "type": "string" }
  },
  "required": ["overview", "supplements"]
}
```

### AC5 — UI components dedicados

**Given** outputs dos agentes de nutrição e suplementação
**When** renderizados no relatório
**Then:**
  - `NutritionPlanView`: tabela de refeições (horário + alimentos + macros), meta semanal, restrições
  - `SupplementPlanView`: lista de suplementos (nome + dosagem + timing + propósito), warnings destacados, data de revisão

**And** ambos registrados no component registry (Story 10.2)
**And** responsivos em 390px

### AC6 — System prompts consideram perfil

**Given** o Medical Profile do paciente
**When** os agentes geram planos
**Then:**
  - "Plano Alimentar" considera: `currentDiet`, `allergies`, `medicalConditions`, `healthObjectives`, `dailyWaterIntake`
  - "Plano de Suplementação" considera: `medications`, `medicalConditions`, `supplementation` (suplementos atuais), `allergies`

### AC7 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Seed: ambos agentes criados com campos corretos
  - Outputs JSON válidos conforme schemas
  - UI components renderizam todos os campos
  - Playwright: layout correto em 390px
