## Story

**Como** paciente,
**Quero** ver um resumo do meu estado de saúde atual ao abrir o app,
**Para** ter uma visão rápida dos meus principais indicadores e alertas.

---

## Contexto

Tela principal do app pós-login. Server Component para fetch inicial (sem loading flash), Client Components para interatividade. Dark mode padrão. Navigation bar mobile na base da tela com ícones para Dashboard, Histórico e Perfil.

---

## Acceptance Criteria

### AC1 — Dados do dashboard

Query para carregar:
1. Último `document` do usuário (por `exam_date DESC`)
2. Snapshot correspondente → `structuredData.modules[].parameters[]` com `status !== 'normal'`
3. Último `complete_analysis` associado → `status`, `createdAt`

Exibe:
- **Card "Última Análise":** data do exame, tipo de documento, data da análise, CTA "Ver relatório"
- **Cards de biomarcadores alterados:** parâmetros com `status` = `high`, `low`, `abnormal`, `borderline` — cada card com nome, valor, unidade, indicador (↑ ↓ ⚠)
- **Status de análise:** se `complete_analysis.status = 'processing'` → badge "Análise em andamento..." com loading spinner
- **Data da última análise** em formato relativo: "há 2 dias"

### AC2 — Estado vazio

- Nenhum exame: card "Envie seu primeiro exame" com ícone + CTA → `/app/upload`
- Exame processado sem análise: card "Análise não iniciada" com CTA "Analisar"
- Análise em processamento: card com spinner "Preparando sua análise..."

### AC3 — CTA fixo

- Botão flutuante "+" ou botão fixo no bottom "Novo exame" → `/app/upload`
- Sempre visível independente do estado

### AC4 — Layout mobile-first

- Cards em coluna única, padding `p-4`
- Elementos interativos mínimo 44px de altura (touch target)
- Shadcn UI: Card, CardHeader, CardContent, Badge, Skeleton, Button
- Dark mode padrão (`className="dark"` no html)
- Bottom navigation bar: Dashboard | Histórico | Perfil (ícones Lucide)

### AC5 — Skeleton loader

- Shadcn `<Skeleton>` durante fetch dos dados
- Suspense boundary no Server Component
- Nunca exibe dados de outro usuário (isolamento por `session.user.id`)

## Notas Técnicas

- Server Component faz fetch com `await auth()` → `session.user.id`
- Biomarcadores alterados: `structuredData.modules.flatMap(m => m.parameters).filter(p => p.status !== 'normal' && p.status !== 'n/a')`
- `'use client'` apenas em componentes com interatividade (botões, hover states)
- Bottom nav como layout component em `src/app/app/layout.tsx` (importa de `src/app/app/components/bottom-nav.tsx`)
- Date relativa: `formatDistanceToNow` do `date-fns` com locale `pt-BR`

### Testing

- **Framework:** Vitest (unit) + Playwright MCP (E2E)
- **Unit:** `tests/unit/dashboard.test.ts` — filtro de biomarcadores por `status !== 'normal'`, isolamento por `userId`
- **E2E:** `tests/e2e/dashboard.spec.ts` — viewport 390px, estado vazio para novo usuário, skeleton visível, bottom nav presente
