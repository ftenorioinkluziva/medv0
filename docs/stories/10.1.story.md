## Story

**Como** paciente,
**Quero** receber um plano de treino personalizado baseado na minha análise,
**Para** ter orientação prática de exercícios adaptada ao meu perfil de saúde.

---

## Contexto

Com Story 9.2 implementada, agentes podem usar `outputType: 'structured'` + `outputSchema` para gerar JSON tipado via `generateObject`. Esta story cria o primeiro agente estruturado: "Plano de Exercícios", que executa na fase specialized e recebe foundation context + Medical Profile (atividades físicas, limitações).

---

## Acceptance Criteria

### AC1 — Health agent "Plano de Exercícios" no seed

**Given** o seed de agentes em `src/lib/db/seed/health-agents.ts`
**When** seed é executado
**Then** existe agente com:
  - `name`: "Plano de Exercícios"
  - `specialty`: "Exercício e Movimento"
  - `analysisRole`: `'specialized'`
  - `outputType`: `'structured'`
  - `outputSchema`: JSON Schema do workout plan (AC2)
  - `systemPrompt`: prompt especializado que orienta geração do plano
  - `isActive`: true

### AC2 — Output schema do workout plan

**Given** o agente "Plano de Exercícios"
**When** `outputSchema` é definido
**Then** o schema inclui:

```json
{
  "type": "object",
  "properties": {
    "overview": { "type": "string" },
    "weeklyGoal": { "type": "string" },
    "workouts": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "day": { "type": "string" },
          "type": { "type": "string" },
          "duration": { "type": "string" },
          "warmup": { "type": "string" },
          "exercises": {
            "type": "array",
            "items": {
              "type": "object",
              "properties": {
                "name": { "type": "string" },
                "sets": { "type": "number" },
                "reps": { "type": "string" },
                "notes": { "type": "string" }
              },
              "required": ["name"]
            }
          },
          "cooldown": { "type": "string" }
        },
        "required": ["day", "type"]
      }
    },
    "restDays": { "type": "array", "items": { "type": "string" } },
    "progressionTips": { "type": "array", "items": { "type": "string" } }
  },
  "required": ["overview", "workouts", "weeklyGoal"]
}
```

### AC3 — Agente executa na fase specialized

**Given** uma análise completa é disparada com o agente "Plano de Exercícios" ativo
**When** a fase specialized roda
**Then** o agente recebe: foundation context + Medical Profile (incluindo `exerciseActivities`, `physicalLimitations`, `exerciseFrequency`, `exerciseDuration`)
**And** gera JSON conforme `outputSchema`
**And** output é persistido em `analyses.content` como JSON stringified

### AC4 — System prompt do agente

**Given** o agente "Plano de Exercícios"
**When** `systemPrompt` é definido
**Then** inclui:
  - Considerar limitações físicas do paciente
  - Adaptar ao nível de atividade atual (frequência, duração, intensidade)
  - Progressão gradual (não escalar volume/intensidade bruscamente)
  - Respeitar condições médicas relevantes
  - Incluir aquecimento e desaquecimento
  - Disclaimer educacional (não substitui personal trainer)

### AC5 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Seed: agente "Plano de Exercícios" criado com campos corretos
  - `analyzeWithAgent` com esse agente: usa `generateObject` (mock confirma schema)
  - Output JSON válido conforme schema
  - Contexto de Medical Profile inclui campos de exercício
