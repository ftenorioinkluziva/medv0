## Story

**Como** paciente,
**Quero** instalar o SAMI no meu celular como um app,
**Para** acessar a plataforma diretamente da tela inicial sem precisar abrir o browser.

---

## Acceptance Criteria

### AC1 — manifest.json completo

- `public/manifest.json` contém: `name`, `short_name`, `description`, `start_url: "/"`, `display: "standalone"`, `orientation: "portrait"`, `background_color`, `theme_color`
- Ícones declarados: `icon-192.png` (192×192) e `icon-512.png` (512×512) com `purpose: "any maskable"`
- **Status:** manifest.json e ícones já existem — validar completude, sem alteração necessária se já conforme

### AC2 — Service Worker ativo

- Service worker registrado e ativo em produção (`/sw.js` ou equivalente)
- Cache de assets estáticos: JS, CSS, fontes (cache-first strategy)
- Não interferir com rotas de API (`/api/*`) — network-only para essas rotas
- Implementação via `next-pwa` (wrapper Workbox para Next.js 16)

### AC3 — Lighthouse PWA score ≥ 80

- Executar Lighthouse audit na rota `/app/dashboard` (ou `/auth/login` se não autenticado)
- Score PWA ≥ 80 (installable + PWA optimized)
- Critérios obrigatórios: HTTPS (Vercel provê), manifest válido, service worker registrado, ícones corretos

### AC4 — Sem regressão no build

- `pnpm build` executa sem erros com `next-pwa` configurado
- `output: 'standalone'` mantido em `next.config.ts`
- Service worker gerado em `public/sw.js` (gitignored ou committed — decisão por configuração)

## Risks

- `next-pwa` pode não ter suporte oficial para Next.js 16 — verificar versão `@ducanh2912/next-pwa` (fork ativo) ou usar `next-pwa@latest`
- `output: 'standalone'` pode conflitar com geração do service worker — testar build local antes de declarar completo
- Lighthouse audit requer HTTPS — testar em preview Vercel ou localhost com `--ignore-certificate-errors`
- Service worker caching pode causar stale content após deploy — configurar `skipWaiting: true` e `clientsClaim: true`

### Configuração next.config.ts

```typescript
import withPWA from '@ducanh2912/next-pwa'
// ou: import withPWA from 'next-pwa'

const nextConfig = {
  output: 'standalone',
  // ... restante da config atual
}

export default withPWA({
  dest: 'public',
  disable: process.env.NODE_ENV === 'development',
  register: true,
  skipWaiting: true,
  // Não cachear rotas de API
  runtimeCaching: [
    {
      urlPattern: /^\/api\//,
      handler: 'NetworkOnly',
    },
  ],
})(nextConfig)
```
