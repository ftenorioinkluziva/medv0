## Story

**Como** paciente,
**Quero** que múltiplos agentes especializados analisem meus exames de forma coordenada,
**Para** receber uma análise completa que conecta diferentes áreas da minha saúde.

---

## Contexto

Engine central do SAMI. Orquestra foundation agents (sequenciais) + specialized agents (paralelos) + síntese. Baseado no `doc/analysis_prompt.md` real do sistema anterior (processo 5 passos + template Markdown). Timeout hard de 60s com cancelamento gracioso por agente via `AbortController`. PII nunca trafega para os LLMs — snapshots já são sanitizados pelo extractor (sem nome, CPF, RG).

### AC3 — `runCompleteAnalysis`

```typescript
// src/lib/ai/orchestrator/complete-analysis.ts
export async function runCompleteAnalysis(
  userId: string,
  documentId: string,
): Promise<{ completeAnalysisId: string; reportMarkdown: string }>
```

Fluxo:
1. Carregar snapshot (`structuredData`) + Medical Profile do usuário (campos clínicos apenas)
2. Carregar foundation agents ordenados por `sort_order` via `getActiveAgentsByRole('foundation')`
3. **Fase 1 — Foundation (sequencial):** cada agente com `AbortController(30s)`; timeout → `status: 'timeout'`, continua com próximo
4. **Fase 2 — Specialized (paralelo):** `Promise.allSettled()` com `AbortController(20s)` por agente
5. **Fase 3 — Síntese:** agente de síntese consolida todos os outputs em relatório Markdown final
6. Persistir `complete_analyses` + atualizar `analyses.complete_analysis_id`

### AC4 — `analyzeWithAgent`

```typescript
// src/lib/ai/agents/analyze.ts
export async function analyzeWithAgent(
  agent: HealthAgent,
  analysisPrompt: string,
  context: {
    snapshotContext: string       // JSON.stringify(snapshot.structuredData) sem PII
    medicalProfileContext: string // campos clínicos do perfil
    knowledgeContext: string      // chunks RAG relevantes
    foundationContext?: string    // outputs dos foundation agents (para specialized)
  },
  signal?: AbortSignal,
): Promise<AgentAnalysisResult>
```

- `generateText` com `system: agent.systemPrompt`, `temperature: Number(agent.temperature)`
- `abortSignal: signal` passado para `generateText`
- RAG: `searchKnowledge(\`${agent.specialty}: ${biomarcadoresAlterados}\`, topK=5)`
- Retorna `{ content, ragContextUsed, tokensUsed, durationMs, status }`

### AC5 — Sanitização de PII

- Snapshot: `patientInfo` do `structuredData` já contém apenas `age` e `gender` (extractor já sanitiza) — verificar e serializar apenas campos não-identificáveis
- Medical Profile: incluir apenas campos clínicos (age, gender, height, weight, pressures, conditions) — excluir qualquer campo de identificação
- Vitest: assert que nenhum payload de LLM contém strings de identificação (nome, CPF, email)

### AC6 — Timeout gracioso

- Foundation agent: `AbortController` com `setTimeout(30_000)` → registrar `status: 'timeout'`, continuar
- Specialized: `Promise.allSettled()` — falhas não cancelam os demais
- Síntese: `AbortController(30_000)`
- Timeout total: 60s hard — retorna análise parcial se atingido

### `AbortController` com timeout

```typescript
const controller = new AbortController()
const timeoutId = setTimeout(() => controller.abort(), 30_000)
try {
  const result = await analyzeWithAgent(agent, prompt, context, controller.signal)
  clearTimeout(timeoutId)
  return result
} catch (error) {
  if (error instanceof Error && error.name === 'AbortError') {
    return { status: 'timeout', content: '', ... }
  }
  throw error
}
```

### Refactoring Performed

- **File:** `src/app/api/analyses/run/route.ts`
  - **Change:** Adicionada verificação de ownership do documento (`documents.userId === session.user.id`) antes de criar o registro de análise. Imports de `and`, `eq` e `documents` adicionados.
  - **Why:** Sem verificação, qualquer usuário autenticado podia disparar análise sobre o documento de outro usuário — os dados médicos do documento alheio seriam processados e os resultados visíveis ao atacante via status endpoint. Violação grave de privacidade de dados médicos.
  - **How:** `db.select({ id: documents.id }).where(and(eq(documents.id, documentId), eq(documents.userId, userId)))` → 404 se não encontrado.

- **File:** `src/lib/ai/orchestrator/complete-analysis.ts`
  - **Change:** Removido UPDATE morto nas linhas 271-274 (`UPDATE analyses SET completeAnalysisId WHERE completeAnalysisId = completeAnalysisId`).
  - **Why:** Todas as inserções em `analyses` já incluem `completeAnalysisId` explicitamente (linha 137). A UPDATE era um no-op total — remanescente de design anterior.
  - **How:** Remoção direta das 5 linhas.

- **File:** `tests/unit/multi-agent-analysis.test.ts`
  - **Change:** Adicionado teste de ownership (404 quando documento não pertence ao usuário). Teste existente "200 válido" atualizado com mock de `db.select` para o novo ownership check.
  - **Why:** Cobrir o fix de autorização com teste explícito.
