## Story

**Como** paciente,
**Quero** ver a evolução dos meus dados de composição corporal,
**Para** acompanhar meu progresso de composição corporal ao longo do tempo.

---

## Contexto

Story 11.2 cria `body_composition_history` e atualiza o perfil. Esta story constrói a UI para visualizar dados atuais e evolução temporal dos indicadores de composição corporal.

---

## Acceptance Criteria

### AC1 — Seção "Composição Corporal" no perfil

**Given** o paciente tem dados de composição corporal no perfil
**When** acessa a tela de perfil (`/app/profile`)
**Then** existe seção "Composição Corporal" exibindo:
  - Peso, % Gordura, Massa Muscular, Gordura Visceral, Massa Óssea, TMB, % Água
  - Data da última atualização
  - Layout compacto em grid 2 colunas (mobile)

**Given** o paciente NÃO tem dados de composição corporal
**When** acessa o perfil
**Then** seção mostra estado vazio: "Envie um exame de bioimpedância para ver seus dados de composição corporal"

### AC2 — Histórico com delta entre medições

**Given** o paciente tem 2+ registros em `body_composition_history`
**When** acessa a seção de composição corporal
**Then** cada indicador mostra delta em relação à medição anterior:
  - "Gordura ↓ 2.1% vs medição anterior"
  - "Massa Muscular ↑ 1.3 kg vs medição anterior"
  - Variação < 1% = "estável" (ou dentro de margem de erro)
  - ↑ para aumento, ↓ para diminuição

**Given** o paciente tem apenas 1 registro
**When** visualiza
**Then** mostra valores atuais sem delta: "Primeira medição — futuras medições mostrarão evolução"

### AC3 — Lista de medições históricas

**Given** o paciente tem múltiplos registros em `body_composition_history`
**When** expande a seção de histórico
**Then** lista ordenada por data (mais recente primeiro)
**And** cada item mostra: data + valores principais (peso, % gordura, massa muscular)
**And** navegação para detalhes de cada medição

### AC4 — Card de composição corporal no dashboard

**Given** o paciente tem dados de composição corporal
**When** acessa o dashboard
**Then** existe card "Composição Corporal" com:
  - Peso atual + delta
  - % Gordura atual + delta
  - Data da última medição
  - Link para perfil completo

**Given** o paciente NÃO tem dados de composição corporal
**When** acessa o dashboard
**Then** card NÃO aparece (sem poluir dashboard)

### AC5 — Responsividade mobile

**Given** todas as UI de composição corporal
**When** visualizadas em 390px
**Then** layout responsivo sem overflow horizontal
**And** cards de indicadores em grid 2 colunas
**And** histórico em lista vertical compacta

### AC6 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - Seção perfil: renderiza com dados, estado vazio sem dados
  - Delta: calcula corretamente entre medições
  - Dashboard card: aparece com dados, não aparece sem
  - Playwright: layout correto em 390px

## Dev Notes

### Delta calculation

```typescript
function calculateDelta(current: number | null, previous: number | null): string | null {
  if (current == null || previous == null) return null
  const diff = current - previous
  if (Math.abs(diff) < 0.5) return 'estável'
  const arrow = diff > 0 ? '↑' : '↓'
  return `${arrow} ${Math.abs(diff).toFixed(1)}`
}
```
