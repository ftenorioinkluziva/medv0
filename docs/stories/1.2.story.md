# Story 1.2 — Authentication + Reset de Senha

**Status:** Done
**Linear:** BLA-37

## Story

**Como** paciente,
**Quero** me registrar, fazer login e recuperar minha senha por email,
**Para** que minha conta seja segura e acessível mesmo se esquecer a senha.

---

## Contexto

NextAuth v5 (Auth.js) com Credentials provider. Reset de senha via Resend (email transacional). Tokens de reset armazenados na tabela `password_reset_tokens` com TTL de 1 hora. Sem OAuth providers no MVP.

---

## Acceptance Criteria

### AC1 — NextAuth v5 configurado
* `next-auth@beta`, `@auth/drizzle-adapter` instalados
* `lib/auth/config.ts` com Credentials provider
* `bcryptjs` para hash de senha no registro e verificação no login
* `app/api/auth/[...nextauth]/route.ts` configurado
* `middleware.ts` protege todas as rotas em `app/(app)/`

### AC2 — Schema `users` completo
* Campos: id (UUID), email (unique), passwordHash, role (enum: patient/admin default patient), onboardingCompleted (bool default false), isActive (bool default true), createdAt, updatedAt
* Tabela `password_reset_tokens`: id, userId (FK), token (UUID único), expiresAt (now + 1h), usedAt (nullable), createdAt

### AC3 — Fluxo de registro
* `POST /api/auth/register` ou Server Action em `app/(auth)/register/`
* Validação Zod: email válido, senha mínimo 8 caracteres
* Erro claro se email já cadastrado
* Redireciona para onboarding após registro bem-sucedido

### AC4 — Fluxo de login
* Tela de login com campos email/senha
* Erro genérico em credenciais inválidas (não expõe se email existe)
* Sessão JWT com `user.id`, `user.email`, `user.role`, `user.onboardingCompleted`
* Redirect pós-login: onboarding se `onboardingCompleted = false`, dashboard se true

### AC5 — Reset de senha
* Tela "Esqueci minha senha" → `POST` com email → gera token → envia email via Resend
* Email com link: `{baseUrl}/auth/reset-password?token={uuid}`
* Tela de nova senha: valida token (não usado, não expirado) → atualiza `passwordHash` → marca token como usado
* Erro claro para token inválido ou expirado

### AC6 — Email via Resend
* `resend` package instalado
* `RESEND_API_KEY` e `RESEND_FROM_EMAIL` em `.env`
* Template do email em `lib/email/templates/reset-password.tsx` (React Email)
* Envio assíncrono — não bloqueia resposta ao usuário

### AC7 — Telas mobile-first
* `/auth/login`, `/auth/register`, `/auth/forgot-password`, `/auth/reset-password`
* Shadcn UI: Card, Form, Input, Button
* Viewport 390px como referência de design

### Security Review

**CRÍTICO — Middleware inativo:** `src/proxy.ts` não é reconhecido como middleware pelo Next.js. Rotas `/app/*` estão desprotegidas. Qualquer usuário não autenticado pode acessar o dashboard diretamente pela URL.

**MÉDIO — Tokens de reset acumulados:** `forgotPasswordAction` não invalida tokens anteriores ao criar novo. Um usuário pode ter N tokens válidos simultâneos para o mesmo email. Recomendado: `UPDATE password_reset_tokens SET usedAt = NOW() WHERE userId = ? AND usedAt IS NULL` antes de inserir novo token.

**OK — Enumeração de email:** `forgotPasswordAction` retorna `success: true` mesmo quando o email não existe. Correto.

**OK — Erro genérico no login:** `verifyCredentials` retorna `null` sem diferenciar "usuário inexistente" de "senha incorreta". Correto.

**OK — Bcrypt rounds:** `bcryptjs.hash(password, 12)` — rounds adequados para produção.

### Performance Considerations

Sem rate limiting nos endpoints de auth. Aceitável para MVP, mas `forgot-password` deve ter rate limiting antes de ir para produção (suscetível a abuso de envio de emails).
