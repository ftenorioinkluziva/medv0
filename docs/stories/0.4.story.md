## Story

**Como** desenvolvedor,
**Quero** Vitest configurado para testes unitários e de integração,
**Para** que as histórias futuras possam validar lógica de negócios com testes rápidos e confiáveis.

---

## Contexto

O projeto já possui `vitest` instalado com configuração parcial. Esta story completa a configuração para garantir que todos os testes unitários passem, que cobertura esteja disponível, e que o setup de jest-dom esteja correto para testes de componentes React futuros.

---

## Acceptance Criteria

### AC1 — Vitest instalado e configurado

* `vitest` instalado como `devDependency`
* `vitest.config.ts` na raiz com:
  * `plugins: [react()]` via `@vitejs/plugin-react`
  * `test.environment: 'node'`
  * `test.globals: true`
  * `test.include: ['tests/**/*.{test,spec}.{ts,tsx}']`
  * `test.exclude: ['tests/e2e/**']`
  * `test.setupFiles: ['./tests/setup.ts']`
  * `resolve.alias: { '@': './src' }`

### AC2 — Setup de testes correto

* `tests/setup.ts` importa `@testing-library/jest-dom`
* `@testing-library/jest-dom` e `@testing-library/react` instalados como `devDependencies`

### AC3 — Scripts npm

* `"test": "vitest run"` — execução única (CI)
* `"test:run": "vitest run"` — alias explícito
* `"test:watch": "vitest"` — modo watch (dev)
* `"test:coverage": "vitest run --coverage"` — cobertura

### AC4 — Cobertura de testes

* `@vitest/coverage-v8` instalado como `devDependency`
* Coverage config em `vitest.config.ts`:
  * `provider: 'v8'`
  * `include: ['src/**/*.{ts,tsx}']`
  * `reporter: ['text', 'lcov']`

### AC5 — Testes unitários passando

* `tests/unit/extractor.test.ts` — 13 testes passando
* `tests/unit/persistence.test.ts` — 8 testes passando
* `tests/unit/auth.test.ts` — testes de auth passando
* `tests/unit/upload.test.ts` — testes de upload passando
* `tests/unit/smoke.test.ts` — smoke test passando
* `pnpm test` passa com 0 falhas

### AC6 — Persistência com transações

* `src/lib/documents/persistence.ts` usa `db.transaction()` para atomicidade
* Ambos os inserts (documents + snapshots) rodam dentro da mesma transação
