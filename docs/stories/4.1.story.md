## Story

**Como** admin,
**Quero** configurar agentes de IA especializados no banco de dados,
**Para** que o sistema possa executar análises por especialidade sem hardcode.

---

## Contexto

Agentes são 100% database-driven — system prompts, modelos e roles configurados no banco, sem hardcode no código. O seed inicial inclui os 5 agentes reais do sistema anterior, com system prompt completo do agente "Medicina Integrativa" como referência (disponível em `doc/system_prompt.md`).

---

## Acceptance Criteria

### AC1 — Schema `health_agents`

Colunas:

* `id` (UUID, PK)
* `name` (text, not null — ex: "Medicina Integrativa")
* `specialty` (text, not null — ex: "Medicina Funcional e Integrativa")
* `description` (text, nullable — descrição pública do agente)
* `system_prompt` (text, not null — prompt completo de identidade e comportamento)
* `analysis_role` (pgEnum: 'foundation' | 'specialized' | 'none')
* `model` (text, default 'google/gemini-2.5-flash' — permite trocar modelo por agente)
* `temperature` (numeric, default 0.7)
* `max_tokens` (integer, nullable — null = model default)
* `is_active` (boolean, default true)
* `sort_order` (integer, default 0 — ordem de execução dos foundation agents)
* `created_at` (timestamp, default now)
* `updated_at` (timestamp)

### AC2 — Seed inicial (5 agentes)

```
1. Medicina Integrativa    → analysisRole: 'foundation', sortOrder: 1
   system_prompt: conteúdo completo de doc/system_prompt.md

2. Endocrinologia          → analysisRole: 'foundation', sortOrder: 2
   system_prompt: adaptar estrutura do doc/system_prompt.md para Endocrinologia

3. Nutrição Clínica        → analysisRole: 'specialized'
4. Medicina do Exercício   → analysisRole: 'specialized'
5. Cardiologia Funcional   → analysisRole: 'specialized'
```

* Seed via `pnpm db:seed` (idempotente — usa `ON CONFLICT (name) DO NOTHING`)

### AC3 — Query helpers

## Dev Notes

* `sort_order` garante ordem de execução determinística para foundation agents
* `model` como text (não enum) — permite adicionar novos modelos sem migration
* System prompt de "Medicina Integrativa" está em `doc/system_prompt.md` — copiar conteúdo completo no seed
* `temperature` como `numeric` (não float) para precisão decimal no Postgres
* Paths reais do projeto: `src/lib/db/schema/`, `src/lib/db/seed/`, `src/lib/db/queries/`

**Tech Debt registrado (SAMI-DEBT-001):**
`updatedAt` não é auto-atualizado em UPDATEs — padrão herdado de Story 3.1. Todo futuro UPDATE path em `health_agents` deve incluir `.set({ updatedAt: new Date() })` explicitamente até que um trigger ou helper global seja implementado.
