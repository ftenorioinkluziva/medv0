## Story

**Como** paciente,
**Quero** um dashboard mais denso e relatórios mais legíveis,
**Para** ver mais informação útil sem rolagem e navegar rapidamente para seções do relatório.

---

## Contexto

Dashboard atual (`src/app/app/dashboard/`) tem cards com espaços vazios em telas pequenas. O relatório não tem sumário ou âncoras — para chegar a "Recomendações" o paciente precisa rolar pelo relatório inteiro. Skeleton loaders podem estar inconsistentes entre telas.

**Componentes do dashboard:** `altered-marker-card.tsx`, `analysis-card.tsx`, `empty-state-card.tsx`, `dashboard-content.tsx`

---

## Acceptance Criteria

### AC1 — Cards compactos no dashboard

**Given** o dashboard em viewport 390px
**When** o paciente visualiza o dashboard
**Then** cards de biomarcadores alterados usam layout compacto:
  - Ícone + nome + valor + delta em uma linha
  - Sem espaço vertical excessivo entre cards
  - Informação densa sem truncamento de dados relevantes

### AC2 — Sumário clicável no relatório

**Given** um relatório de análise completo com múltiplas seções
**When** o paciente abre o relatório
**Then** existe um sumário no topo com links âncora para cada seção (Resumo Executivo, Eixos, Padrões, Insights, Recomendações)
**And** clicar em um link rola suavemente (`scroll-behavior: smooth`) até a seção correspondente
**And** cada heading de seção tem um `id` para âncora

### AC3 — Skeleton loaders consistentes

**Given** qualquer tela em estado de carregamento
**When** dados estão sendo buscados
**Then** skeleton loaders são exibidos com formato que reflete o layout final
**And** skeletons usam `Skeleton` do Shadcn UI consistentemente

**Telas que precisam de skeleton:**
- Dashboard: cards de biomarcadores + card de análise
- Relatório: área de Markdown
- Perfil: campos do formulário
- Histórico: lista de documentos

### AC4 — Transições suaves entre telas

**Given** o paciente navega entre telas via bottom nav ou links
**When** a navegação acontece
**Then** não há flash branco/preto entre telas
**And** loading state é exibido durante transição (React Suspense + skeleton fallback)

## Dev Notes

### Sumário do relatório — approach

O relatório é Markdown parseado. A geração do sumário pode:
1. Parsear headings `##` do Markdown antes de renderizar
2. Gerar `<nav>` com links para `#secao-nome`
3. Adicionar `id` slugificado a cada heading no render

```typescript
// Extrair headings
const headings = markdown.match(/^## .+$/gm)?.map(h => ({
  text: h.replace(/^## /, '').replace(/[^\w\s]/g, '').trim(),
  id: slugify(h),
}))
```

### Dashboard cards — layout target

```
┌──────────────────────────┐
│ ↑ LDL  182 mg/dL  +12%  │
│ ↓ VitD  18 ng/mL  -8%   │
│ ⚠ Glicose  99 mg/dL  =  │
└──────────────────────────┘
```

Uma linha por biomarcador, ícone de status + nome + valor + delta.

### Suspense pattern

```tsx
// src/app/app/dashboard/page.tsx
import { Suspense } from 'react'
import { DashboardSkeleton } from './components/dashboard-skeleton'

export default function DashboardPage() {
  return (
    <Suspense fallback={<DashboardSkeleton />}>
      <DashboardContent />
    </Suspense>
  )
}
```
