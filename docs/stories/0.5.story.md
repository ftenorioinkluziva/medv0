## Story

**Como** desenvolvedor,
**Quero** Playwright configurado via MCP para testes E2E com viewport mobile,
**Para** que as histórias futuras possam verificar fluxos completos no browser com foco mobile.

---

## Contexto

O SAMI é mobile-first (viewport 390px). Os testes E2E usam o MCP do Playwright (já configurado no Claude Code) para automação interativa durante desenvolvimento. Também existe uma configuração standalone (`@playwright/test`) para execução autônoma em CI.

**Dois modos de uso:**
1. **MCP (desenvolvimento)**: Claude Code usa `mcp__plugin_playwright_playwright__*` — automação interativa, não requer `playwright.config.ts`
2. **Standalone (CI)**: `pnpm test:e2e` — execução autônoma com `playwright.config.ts`

---

## Acceptance Criteria

### AC1 — Instalação Playwright standalone
* `@playwright/test` instalado como `devDependency`
* `playwright.config.ts` na raiz do projeto
* Browser: `chromium` apenas para CI (suficiente, menor footprint)
* `pnpm test:e2e` executa os testes E2E no modo standalone

### AC2 — Configuração mobile-first correta
* Usar `devices['iPhone 14 Pro']` do Playwright como base do projeto
* **NÃO** duplicar `viewport` manualmente — `devices['iPhone 14 Pro']` já inclui:
  * `viewport: { width: 390, height: 844 }`
  * `isMobile: true`
  * `deviceScaleFactor: 3`
  * User agent mobile correto
* `use` global no config: apenas `baseURL`, `screenshot: 'only-on-failure'` e `trace: 'on-first-retry'`

### AC3 — Configuração de ambiente e CI
* `baseURL: process.env.PLAYWRIGHT_BASE_URL || 'http://localhost:3000'`
* Timeout por teste: 30 segundos (`timeout: 30_000`)
* Retries: `process.env.CI ? 2 : 0`
* `webServer` block para CI — inicia o dev server automaticamente

### AC4 — Estrutura de diretórios
* `tests/e2e/` — specs E2E
* `tests/e2e/fixtures/` — fixtures reutilizáveis
* `playwright-report/`, `test-results/` no `.gitignore`
* `tests/e2e/README.md` explicando diferença entre uso via MCP e standalone

### AC5 — Smoke test funcionando
* `tests/e2e/smoke.spec.ts`: abre `http://localhost:3000` e verifica que a página carrega sem erros JS
* Teste usa `devices['iPhone 14 Pro']` como viewport
* `expect(page).toHaveTitle(/.+/)` e `expect(page.locator('body')).toBeVisible()`

### AC6 — Integração CI
* Script `"test:e2e": "playwright test"` no `package.json`
* Script `"test:e2e:ci": "playwright test --reporter=github"` para CI
* `.gitignore` atualizado: `playwright-report/`, `test-results/`
