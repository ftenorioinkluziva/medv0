## Story

**Como** paciente,
**Quero** receber um relatório claro e compreensível sobre minha saúde,
**Para** entender meus resultados sem precisar de conhecimento médico.

---

## Contexto

O relatório é o produto final visível ao paciente. Gerado pela fase de síntese do motor multi-agente (Story 4.2) como Markdown estruturado. Disclaimer obrigatório injetado programaticamente — não dependente do LLM. Template baseado no `doc/analysis_prompt.md` real do sistema anterior, **sem** `[Nome do Paciente]` (LGPD).

## Acceptance Criteria

### AC1 — Template Markdown obrigatório

O relatório de síntese **deve** seguir este template (seções obrigatórias):

```markdown
# Análise Integrativa de Saúde

## 📋 Resumo Executivo
[Visão geral em 2-3 parágrafos dos principais achados e correlações]

## 🔍 Análise Detalhada por Eixos Funcionais

### Eixo Tireoidiano e Energético
- **Biomarcador:** [Valor] (Ref. Lab: [X-Y] | **Alvo Funcional: [valor]**)
  - **Interpretação:** [análise]

### Saúde Metabólica e Inflamatória
[...]

### Status Nutricional e Hematológico
[...]

### Função Detox e Imunológica
[...]

## ⚠️ Padrões e Pontos de Atenção
- **[Padrão]:** [Descrição com indicadores ↑ ↓ ⚠]

## 💡 Insights e Hipóteses de Causa Raiz
[Hipóteses sobre causas raiz — sempre com "sugere", "indica", nunca diagnóstico]

## 📚 Recomendações Educacionais
1. **[Área]:** [Sugestão educacional com citação RAG quando disponível]
   - Fonte: [título do artigo]

## ⚕️ Disclaimer
> Esta análise é gerada por IA para fins educacionais e NÃO substitui consulta médica profissional. Consulte sempre um médico qualificado.
```

**Nota:** O título usa "Análise Integrativa de Saúde" — sem nome do paciente (LGPD).

### AC2 — Disclaimer obrigatório

- Injetado programaticamente ao final do relatório, após a geração do LLM
- Texto exato: `"Esta análise é gerada por IA para fins educacionais e NÃO substitui consulta médica profissional"`
- Não pode ser omitido pelo LLM — código garante presença antes de persistir

### AC3 — Biomarcadores com indicadores visuais

- Status `high` ou `abnormal` → prefixo `↑`
- Status `low` → prefixo `↓`
- Status `borderline` → prefixo `⚠`
- Indicadores inseridos pelo agente de síntese com base nos dados do snapshot

### AC4 — Citações do RAG

- Quando RAG contextualiza uma análise: `Fonte: [título do artigo] — [autor]`
- Citações no final de cada recomendação quando fundamentada em knowledge base
- Sem citações inventadas — apenas artigos realmente usados em `context.knowledgeContext`

### AC5 — Persistência

- `complete_analyses.report_markdown` armazena o relatório completo
- Relatório acessível via `GET /api/analyses/[completeAnalysisId]`
- **Auth obrigatória:** verificar `completeAnalysis.userId === session.user.id` → 403 se não corresponder
- Retorna `{ reportMarkdown, createdAt, agentsCount }`
- `agentsCount` exibido na UI como "Analisado por X especialistas"

### AC6 — Renderização na UI (mobile-first)

- Tela `/app/analyses/[id]` renderiza o Markdown
- **Usar AI Elements** `<MessageResponse>` para renderizar o Markdown — não `<p>{text}</p>` raw
- Consultar `npx ai-elements` para componentes disponíveis
- Seções colapsáveis opcionais para relatórios longos
- Scroll vertical fluido no viewport 390px
- **Loading state:** Shadcn `<Skeleton>` enquanto fetch do relatório
- **Error state:** mensagem "Relatório não encontrado" se 404, "Acesso negado" se 403

## Notas Técnicas

- Prompt de síntese instrui o modelo a seguir o template **exatamente** — seções nomeadas com emojis como âncoras
- Disclaimer injetado DEPOIS da geração: `reportMarkdown + '\n\n---\n\n> ' + DISCLAIMER_TEXT`
- Verificação de seções: função utilitária `validateReportSections(markdown)` retorna quais seções faltam — falha silenciosa (loga warning, não lança erro)
- AI Elements `<MessageResponse>` renderiza Markdown com syntax highlighting, links, tables — ideal para relatórios médicos
- `[Nome do Paciente]` removido do template (LGPD) — título fixo "Análise Integrativa de Saúde"
- Consultar Context7: AI Elements, AI SDK v6 renderização de texto
