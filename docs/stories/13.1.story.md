## Story

**Como** paciente,
**Quero** uma experiência mobile com menos rolagem e navegação mais fluida,
**Para** encontrar funcionalidades rapidamente e consumir informações sem rolagem excessiva.

---

## Contexto

O SAMI é mobile-first (PWA). Observações do uso real (v1.0):
- Rolagem excessiva nas telas de relatório e perfil
- Navegação depende do header — bottom nav bar é padrão mobile para acesso rápido
- Font sizes e spacing podem ser otimizados para telas 390px+
- `AnalysisStatusCard` (`src/app/app/analyses/[id]/analysis-status-card.tsx`) tem strings com acentuação possivelmente incorreta

**Viewport target:** 390px (iPhone 14/15 base)

---

## Acceptance Criteria

### AC1 — Bottom navigation bar

**Given** navegação atual depende do header
**When** o paciente está em qualquer tela `/app/*`
**Then** existe uma bottom nav bar fixa com 4 itens: Dashboard, Upload, Histórico, Perfil
**And** item ativo é destacado visualmente
**And** a nav bar não aparece em telas de auth ou admin

### AC2 — Font sizes otimizados para mobile

**Given** textos em viewport mobile (390px)
**When** uma auditoria de font sizes é feita
**Then:**
  - Body text: 14-15px (0.875-0.9375rem)
  - Headings h1: 1.5rem, h2: 1.25rem, h3: 1.125rem
  - Captions/labels: 0.75-0.8125rem
**And** line-height confortável (1.5 para body, 1.25 para headings)

### AC3 — Padding/margin reduzidos em cards

**Given** cards com padding excessivo em mobile
**When** espaçamento é otimizado
**Then** padding interno de cards: `p-3` (12px) em vez de `p-4`/`p-6`
**And** gap entre cards: `gap-3` em vez de `gap-4`/`gap-6`
**And** margens laterais da página: `px-4` (16px) consistente

### AC4 — Seções colapsáveis no relatório

**Given** o relatório da análise tem múltiplas seções longas
**When** o paciente visualiza o relatório
**Then** seções do relatório (Eixos Funcionais, Padrões, Insights, Recomendações) são colapsáveis
**And** Resumo Executivo e Evolução começam expandidos
**And** demais seções começam colapsadas
**And** collapse usa Accordion do Shadcn UI

### AC5 — Correção de acentuação em AnalysisStatusCard

**Given** `src/app/app/analyses/[id]/analysis-status-card.tsx` pode ter strings com problemas de encoding
**When** o componente é auditado
**Then** todas as strings estão com acentuação correta em pt-BR

### AC6 — Viewport mobile sem horizontal overflow

**Given** todas as telas em viewport 390px
**When** auditadas via Playwright
**Then** nenhuma tela apresenta scroll horizontal (`overflow-x` indesejado)
**And** nenhum elemento extrapola a largura da viewport
