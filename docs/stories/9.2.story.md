## Story

**Como** admin,
**Quero** configurar parâmetros avançados de modelo e schema de output por agente,
**Para** ter controle fino sobre o comportamento de cada agente e habilitar outputs estruturados (JSON tipado) para agentes especializados.

---

## Contexto

Atualmente `health_agents` tem apenas `model`, `temperature` e `maxTokens`. O PRD v1.1 requer:
- `modelConfig` JSONB para parâmetros avançados (topP, topK, frequencyPenalty, etc.)
- `outputSchema` JSONB para JSON Schema (quando agente deve gerar objeto tipado)
- `outputType` TEXT para controlar se agente gera texto ou objeto estruturado

`analyzeWithAgent()` usa apenas `generateText()` — precisa bifurcar para `generateObject()` quando `outputType === 'structured'`.

---

## Acceptance Criteria

### AC1 — Schema fields adicionados

**Given** tabela `health_agents` sem modelConfig/outputSchema/outputType
**When** migration é aplicada
**Then** existem campos:
  - `modelConfig` JSONB nullable — `{ topP?, topK?, frequencyPenalty?, presencePenalty?, seed? }`
  - `outputSchema` JSONB nullable — JSON Schema para `generateObject`
  - `outputType` TEXT NOT NULL DEFAULT `'text'` — `'text'` | `'structured'`

**And** agentes existentes ficam com `outputType = 'text'`, `modelConfig = null`, `outputSchema = null`

### AC2 — `analyzeWithAgent` bifurcação por outputType

**Given** agente com `outputType: 'text'`
**When** `analyzeWithAgent()` é chamado
**Then** usa `generateText()` com comportamento atual (retorna Markdown)

**Given** agente com `outputType: 'structured'` e `outputSchema` definido
**When** `analyzeWithAgent()` é chamado
**Then** usa `generateObject()` com o schema do agente
**And** retorna JSON stringified no campo `content` do `AgentAnalysisResult`
**And** `AgentAnalysisResult` ganha campo opcional `structuredOutput: unknown` com o objeto parseado

### AC3 — `modelConfig` aplicado ao generate

**Given** agente com `modelConfig: { topP: 0.9, seed: 42 }`
**When** `analyzeWithAgent()` é chamado
**Then** `topP` e `seed` são passados como opções ao `generateText`/`generateObject`
**And** `temperature` do agente continua sendo respeitado
**And** `modelConfig` nulo → parâmetros default do provider

### AC4 — Validação de outputSchema

**Given** admin salva um agente com `outputType: 'structured'`
**When** `outputSchema` está vazio ou inválido (não é JSON Schema válido)
**Then** form exibe erro: "Output Schema obrigatório para agentes estruturados"

**Given** admin salva um agente com `outputType: 'text'`
**When** `outputSchema` está vazio
**Then** validação passa (schema não é necessário para text)

### AC5 — Admin form: editor de modelConfig e outputSchema

**Given** a tela de edição/criação de agente
**When** admin edita o agente
**Then:**
  - Seção "Configuração Avançada" com campos numéricos para topP, topK, frequencyPenalty, presencePenalty, seed
  - Toggle `outputType`: "Texto" | "Estruturado"
  - Quando "Estruturado" selecionado: textarea para JSON Schema com validação em tempo real
  - Preview do JSON Schema renderizado

### AC6 — Testes

**Given** as mudanças
**When** testes são executados
**Then:**
  - `analyzeWithAgent` com `outputType: 'text'`: usa `generateText` (mock confirma)
  - `analyzeWithAgent` com `outputType: 'structured'` + schema: usa `generateObject` (mock confirma)
  - `modelConfig` passado corretamente ao generate call
  - Schema validation: rejeita structured sem outputSchema
  - Agentes existentes (sem novos campos): comportamento idêntico

### `generateObject` com JSON Schema dinâmico

O Vercel AI SDK suporta `jsonSchema()` helper para schemas dinâmicos (não Zod). Verificar documentação atual para a assinatura exata.

### modelConfig spread

```typescript
function spreadModelConfig(config: ModelConfig | null) {
  if (!config) return {}
  const result: Record<string, unknown> = {}
  if (config.topP != null) result.topP = config.topP
  if (config.topK != null) result.topK = config.topK
  if (config.seed != null) result.seed = config.seed
  // frequencyPenalty/presencePenalty — verificar se AI SDK aceita diretamente
  return result
}
```

## Scope

**IN:** Campos modelConfig/outputSchema/outputType no schema, bifurcação generateText/generateObject, admin form com editor JSON
**OUT:** Validação semântica do outputSchema (apenas sintaxe JSON), UI de preview de output, versioning de schemas

## Complexity

**L** (Large) — Schema change + bifurcação no core AI + admin form complexo (JSON editor). ~5h estimado.

## Risks

- **MEDIUM:** `generateObject` com JSON Schema dinâmico pode ter edge cases de parsing. Mitigação: fallback para `generateText` se `generateObject` falha.
- **LOW:** Admin pode inserir schema inválido — validação sintática cobre maioria dos casos.
