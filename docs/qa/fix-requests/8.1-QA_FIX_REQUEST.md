### 1. [HIGH] TEST-001 — 3 cenários P0 de `searchKnowledge` com `agentId` ausentes

**Issue ID:** FIX-8.1-001

**Location:** `tests/unit/vector-search.test.ts`

**Problema:**

`searchKnowledge` foi refatorada para aceitar `agentId?` opcional com filtro SQL via UNION subquery
(`vector-search.ts:154-160`). A lógica está correta, mas `vector-search.test.ts` não tem nenhum
teste que passe `agentId`. Os 3 cenários de AC3 (B, C, D) não têm cobertura.

Se alguém remover ou quebrar o `agentScopeFilter` em um refactor futuro, nenhum teste detectará.

**Cenários faltantes:**
- **Cenário B** — com `agentId`, artigo associado ao agente → deve retornar chunks
- **Cenário C** — com `agentId`, artigo global (`isGlobal=true`, não associado) → deve retornar chunks
- **Cenário D** — com `agentId`, artigo de outro agente (`isGlobal=false`) → **NÃO** deve retornar

**Expected — adicionar este bloco em `tests/unit/vector-search.test.ts`:**

Adicionar após o describe existente `'query sem match retorna array vazio'`, antes do
describe `'incrementUsageCount'`:

```typescript
describe('searchKnowledge com agentId (filtro por agente)', () => {
  it('Cenário B: com agentId retorna chunks de artigo associado ao agente', async () => {
    // #given
    const chain = buildSelectChain([mockRows[0]])
    const agentId = 'agent-nutricao-uuid'

    // #when
    const results = await searchKnowledge('dieta', 5, agentId)

    // #then — WHERE filter aplicado e artigo associado retornado
    expect(chain.where).toHaveBeenCalledOnce()
    expect(results).toHaveLength(1)
    expect(results[0].articleId).toBe('article-1')
  })

  it('Cenário C: com agentId retorna chunks de artigo global (isGlobal=true) mesmo sem associação explícita', async () => {
    // #given — DB retorna artigo global (simulando UNION is_global=true)
    const chain = buildSelectChain([mockRows[1]])
    const agentId = 'agent-nutricao-uuid'

    // #when
    const results = await searchKnowledge('integrativa', 5, agentId)

    // #then — artigo global acessível por qualquer agente
    expect(chain.where).toHaveBeenCalledOnce()
    expect(results).toHaveLength(1)
    expect(results[0].articleId).toBe('article-2')
  })

  it('Cenário D: com agentId NÃO retorna chunks de artigo de outro agente (isGlobal=false)', async () => {
    // #given — DB retorna vazio pois filtro exclui artigo de outro agente
    const chain = buildSelectChain([])
    const agentId = 'agent-nutricao-uuid'

    // #when
    const results = await searchKnowledge('exercício', 5, agentId)

    // #then — isolamento de escopo garantido
    expect(chain.where).toHaveBeenCalledOnce()
    expect(results).toHaveLength(0)
  })

  it('sem agentId não aplica filtro (backward compat — Cenário A)', async () => {
    // #given
    const chain = buildSelectChain(mockRows)

    // #when
    const results = await searchKnowledge('vitamina D')

    // #then — where chamado com undefined = sem filtro de escopo
    expect(chain.where).toHaveBeenCalledWith(undefined)
    expect(results).toHaveLength(3)
  })
})
```

**Nota sobre o 4º teste (Cenário A):** O describe existente já cobre backward compat
funcionalmente, mas o teste explícito de `where(undefined)` documenta o contrato de forma
inequívoca. Opcional se o @dev preferir não duplicar.

## Context

**Por que estes testes são importantes:**

O filtro `agentScopeFilter` em `vector-search.ts:154-160` usa uma UNION subquery SQL:
```sql
WHERE ke.article_id IN (
  SELECT ak.article_id FROM agent_knowledge ak WHERE ak.agent_id = $agentId
  UNION
  SELECT kb.id FROM knowledge_base kb WHERE kb.is_global = true
)
```

Esta é a lógica central da Story 8.1 — sem testes para os Cenários B, C e D, qualquer
refactor que acidentalmente remova ou altere este filtro passaria despercebido pelo CI.
